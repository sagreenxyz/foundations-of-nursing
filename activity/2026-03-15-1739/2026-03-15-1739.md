## Part 1: Sprint Goals

1. **Primary goal** — What is the single most important outcome you want to achieve by the end of this sprint?

   Write the full first draft of Chapter 1 (*History and Evolution of Nursing*) in markdown format and make it visible in a working web-page application so the developer can always see what the published output will look like.

2. **Secondary goals** — Are there additional deliverables you want to complete this sprint?

   - Establish the chapter template (backlog item 0.5) so all future chapters follow a consistent structure.
   - Build the web application foundation (`docs/` directory with navigation, styling, and markdown rendering) so it can be extended as each new chapter is drafted.

3. **Definition of done** — How will you know this sprint is complete?

   - Chapter 1 full draft exists as a well-structured markdown file at `content/chapter-01.md`.
   - A web-page application hosted from the `docs/` directory renders the chapter content clearly, with navigation, styling appropriate for an academic textbook, and all chapter elements (objectives, key terms, case study, NCLEX questions) visible.
   - The developer can open `docs/index.html` locally in a browser and immediately see the rendered output.

## Part 3: Web Application

7. **Hosting target** — Where will the web application be published?

   GitHub Pages from the `docs/` directory. The application should work both locally (opened as a file) and when deployed on GitHub Pages.

8. **Navigation scope** — Should the navigation reflect all planned chapters now, or just Chapter 1?

   Show all chapters from the merged TOC in the navigation sidebar. Mark chapters as "Draft Available," "Outline Only," or "Coming Soon" to indicate their current status at a glance.

9. **Styling requirements** — What visual style should the web application use?

   Clean, academic, readable. Serif body text (Georgia or similar), adequate line-height, generous padding. The goal is readability and a textbook feel, not a marketing site.
